# Widget authoring guide

This guide has been merged into a single, self-contained, AI-ready document:

→ **[widgets.md](widgets.md)** — *Writing a widget for Widget Dashboard*

That file is everything needed to build a working widget: folder layout, the
full manifest reference, the backend (`WidgetBase` + `ctx`) and frontend
(`mount` + `api`) contracts, host services, events/triggers, the rules that
matter (with the bugs they prevent), install/test/package steps, an authoring
checklist, and a complete copy-paste example.

Hand `widgets.md` to an AI along with a description of the widget you want — it
needs nothing else.
